# HTML試作品仕様書

> この仕様書は、現行のHTML試作品（`index.html` / `style.css` / `app.js`）の実装内容に合わせて更新している。当初の設計から変わった点は本文中で「現行プロトタイプ」として明記する。

## 1. 目的

公開サイトと管理画面の見た目を、実装前に確認するためのHTML試作品を作成する。

初期試作品では、GAS連携や実データ保存は行わず、HTML/CSS/JavaScriptのみで以下を確認できる状態を目指す。

- 公開サイトの雰囲気
- スマホでの読みやすさ
- 講座カードの見え方
- 活動レポート記事の見え方
- 募集カードの見え方
- 管理画面の操作感
- 左入力・右スマホプレビューの使いやすさ
- 写真が少なくても成立するか
- ChatGPTコピペ運用のイメージ

## 2. ファイル構成案

```text
unnan-learning-site/
├─ index.html
├─ style.css
├─ app.js
└─ README.md
```

初期は1画面完結でもよい。画面切替はJavaScriptで行う。

将来的に分割する場合：

```text
unnan-learning-site/
├─ public.html
├─ admin.html
├─ style.css
├─ app.js
├─ sample-data.js
└─ README.md
```

## 4. デザイン方針

### 全体

- 白基調
- 余白多め
- カードは角丸
- 文字は読みやすく
- 写真に依存しすぎない
- 講座ごとに差し色
- 小さな手描き風線画アイコンを使う

### 基本色（現行プロトタイプのCSS変数）

```text
--bg:         #fbf7ef   ページ背景（生成り）
--paper:      #fffdf8   カード・紙面
--paper-soft: #f8f1e7   淡い面
--ink:        #2d2924   本文テキスト
--text-soft:  #5e554d   やや弱いテキスト
--muted:      #8a7f73   メタ・補助テキスト
--warm-brown: #9b6a47   綴じの差し色・リンク基調
```

### 講座差し色（3トーン）

旧設計の単色から、講座ごとに「主色（color）／淡い背景色（soft）／文字色（ink）」の3トーンへ変更し、全体に彩度を落とした落ち着いた配色にしている。

```text
おしゃべりクッキング: color #c9824a / soft #f5e4d5 / ink #9f5f35
華凜:               color #b9788e / soft #f4e4e9 / ink #965f72
アグリスクール:       color #7f9b73 / soft #e8efe3 / ink #5e7a55
赤ちゃん食堂:         color #d6b766 / soft #f6edcf / ink #9f8140
ヨガ講座:            color #7fa8a0 / soft #e4efed / ink #5d827b
```

カードやラベルは `--course-color` `--course-soft` `--course-ink` として各講座色を流し込む。

## 6. 公開サイト機能

### 画面切替

現行プロトタイプはハッシュルーティング（`#home` など）のSPAで、以下に切り替える。

- トップ（home）
- 講座一覧（courses）
- 各講座ページ（course-{id}）
- 活動レポート一覧（reports）
- 活動レポート詳細（article）
- 募集案内詳細（recruit）
- お問い合わせ（contact）
- 管理画面（admin）

### 募集カード表示

- 募集データがあり、締切日が現在日以降の場合のみトップに表示
- 試作品では現在日判定を簡易的にしてよい
- 募集がない状態を切り替えるデモボタンがあるとよい

### 活動レポート一覧

- 最新3件をトップに表示
- 一覧ページでは講座フィルターで絞り込み（すべて＋各講座）
- キーワード検索（タイトル・内容）を実装済み
- 年の絞り込み（記事の開催年から自動生成）を実装済み。旧設計では将来機能としていたが前倒しした

### 記事詳細

- 講座名ラベル
- 令和日付
- タイトル
- 講師情報
- 本文
- 写真枠
- 今回の内容
- 参加者の声
- 同じ講座の過去レポート

写真がない場合は、淡い背景のプレースホルダーを表示する。

## 7. 管理画面機能

### ダッシュボード

- 新規記事作成ボタン
- 過去記事登録ボタン
- 募集案内作成ボタン
- 下書き一覧
- 公開予約一覧
- 最近公開した記事
- 小さなスマホプレビュー

### 記事一覧

- 表形式表示
- カード形式表示切替
- 状態ラベル
- 編集ボタン

### 記事作成

左入力、右スマホプレビュー。

入力項目：

- 投稿タイプ
- 講座名
- 開催日
- タイトル
- 講師名
- 講師肩書き
- 講師Instagram
- 内容メモ
- 本文
- 写真アップロードUI
- メニュー・制作物
- 参加者の声
- Instagram用本文
- ハッシュタグ
- 公開状態
- 公開予約日時
- 写真掲載チェック

### スマホプレビュー（Phone View）

入力内容をスマホプレビューに反映する。現行プロトタイプでは、このプレビューを画面上をドラッグで動かせるフローティングパネル「Phone View」として実装している（macOSのウィンドウ風に、閉じる・最小化・最大化ができる）。固定の右カラムではない。

### 写真管理UI

試作品では実ファイル保存は不要だが、以下を作り込んでいる。

- 撮影ガイドの常時表示
- 写真の複数選択アップロード
- 「掲載イメージ（公開サイトと同じトリミング）／元写真」の切り替え表示
- 写真ごとのカード：使用可否・役割（メイン／本文中／ギャラリー／使わない）・トリミング位置（中央／上／下／左／右）・写真掲載チェック・キャプション・altテキスト・AIアドバイスメモ
- 「AIに写真を相談するプロンプトをコピー」ボタン（記事情報を差し込んだ写真相談用プロンプトを生成）
- AIからの写真アドバイスを貼り付ける欄（自動反映はせず手動設定）

### ChatGPT用メモコピー

入力内容からChatGPT用メモを生成し、コピーできるようにする。写真については、上記の写真相談用プロンプトも別途コピーできる。試作品では「コピーしました」表示まででよい。

### 公開前チェック（3点必須）

以下の3点すべてにチェックが入るまで、公開ボタンを押せないようにする（旧設計の1点から拡張）。

```text
□ 写真掲載に問題がないことを確認しました
□ 顔や個人情報が写りすぎていないことを確認しました
□ 必要に応じて掲載許可を確認しました
```

## 8. レスポンシブ仕様

### スマホ

- 1カラム
- カードは縦並び
- 管理画面は入力／プレビューをタブ切替
- ヘッダーはシンプル
- ハンバーガーメニュー

### PC

- 公開サイトは最大幅1100px前後
- 管理画面は左入力・右プレビューの2カラム
- 記事一覧は表形式を基本

## 10. HTML試作品で確認したいこと

- スマホで読みやすいか
- 写真なしでも見た目が崩れないか
- 講座ごとの違いが差し色で分かるか
- 募集感が強すぎないか
- 活動レポートが主役に見えるか
- 管理画面が職員にとって難しそうに見えないか
- ChatGPTコピペ運用が自然か
- 過去記事を短く登録しても成立するか
