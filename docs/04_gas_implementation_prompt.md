# GAS実装用プロンプト

## 使い方

このMarkdownは、CodexやChatGPTにGAS実装を依頼するためのプロンプトです。  
実装時は、この内容をそのまま貼り付けて、段階的に開発してください。

---

# 依頼内容

JAしまね雲南地区本部の講座・活動記録サイト「うんなん暮らしの学び帖」を、Google Apps Script（GAS）＋Googleスプレッドシート＋Googleドライブで構築してください。

## 1. サイトの目的

本サイトは、JAしまね雲南地区本部が実施する以下の講座・活動を、地域の方にわかりやすく伝え、活動レポートとして蓄積するためのWebサイトです。

単なる募集サイトではなく、講座の雰囲気や活動内容をやわらかく残す「小さな広報メディア」として運用します。

## 2. 対象講座

### メイン講座

1. おしゃべりクッキング  
   コピー：食べて、話して、暮らしを楽しむ料理教室

2. 華凜  
   コピー：学びと交流を重ねる、大人の女性のための時間

3. アグリスクール  
   コピー：親子で食と農にふれる、わくわく体験教室

4. 赤ちゃん食堂  
   コピー：赤ちゃんと保護者が、ほっとできる食と交流の場

### その他の講座

5. ヨガ講座  
   コピー：からだをゆるめて、心を整えるやさしい時間

## 5. 公開サイト仕様

### トップページ

表示順（現行プロトタイプに合わせる）：

1. ヘッダー（ホーム／活動レポート／講座一覧／Instagram／お問い合わせ）
2. カバー（導入小見出し・リード・説明文と、最新の活動レポートを大きく表示）
3. 最近の活動レポート3件（募集がある時は「次回のご案内」を控えめに横へ添える）
4. 講座一覧（メイン4講座と、その他の講座を1つの縦リストで表示）
5. 季節の活動カード（春夏秋冬）
6. フッター

サイト名：

```text
JAしまね雲南地区本部
うんなん暮らしの学び帖
```

導入・トップコピー：

```text
JAでひらく、講座と活動の記録。

食べて、つくって、話して。
暮らしを少し楽しく。

料理教室、JA女性大学、親子体験、赤ちゃん食堂など。
雲南の暮らしに寄り添う小さな活動を、写真とことばで記録しています。
```

### 次回のご案内（募集）

募集記事のうち、申込締切日が現在日以降で、状態が公開のもののみ表示してください。トップページでは大きなバナーにせず、最近の活動レポートの横に「次回のご案内」として控えめに表示してください。

募集がない場合は、このセクション自体を非表示にしてください。

### 最新レポート

投稿タイプが「活動レポート」で状態が「公開」の記事を、開催日または公開日順に新しい順で3件表示してください。

### 講座カード

4つのメイン講座を表示してください。  
写真がなくても成立するよう、差し色・アイコン・コピーで見せてください。

### 季節の活動カード

春夏秋冬の4カードを表示してください。

内容：

```text
春：料理教室や、暮らしを楽しむ学びの時間。
夏：親子で食と農にふれる体験や、赤ちゃん食堂。
秋：季節の食材を味わう料理教室や交流の時間。
冬：手しごと、発酵食品、あたたかな食卓づくり。
```

## 6. 講座ページ仕様

各講座ページには以下を表示してください。

- 講座名
- 説明コピー
- 講座紹介文
- 差し色
- アイコン
- 次回予定・募集リンク（該当募集がある時のみ）
- 同じ講座の過去レポート
- Instagramリンク

写真なしでも成立するデザインにしてください。良い写真がある場合のみメイン写真として使います。

- 年の絞り込み（記事の開催年から自動生成）を設置。旧設計では将来機能としていたが、現行プロトタイプで実装済み

講座フィルター：

- すべて
- おしゃべりクッキング
- 華凜
- アグリスクール
- 赤ちゃん食堂
- ヨガ講座

## 8. 記事詳細仕様

活動レポート詳細には以下を表示してください。

- 「活動レポート」ラベル・講座名ラベル
- タイトル
- サブタイトル（リード見出し）
- メタ情報（開催日（令和表記）｜開催場所｜講師）
- コピー／共有ボタン（リンクをコピー／紹介文つきでコピー）
- メイン写真
- 本文（段落）
- 今回の内容（メニュー・制作物・体験内容をまとめたリスト）
- 参加者の声
- この講座のほかの記録（同じ講座の過去レポート）

講師は、入力では名前・肩書きを分けて持ち、表示では「肩書き＋名前」を1行にまとめて見せてください。

必須項目：

- 講座名
- 開催日
- タイトル
- 本文または内容メモ
- 写真1枚以上

ただし、過去記事登録モードでは本文が短くても記事として成立するようにしてください。

## 9. 募集記事仕様

募集記事は、募集がある時だけ作成します。

- トップページには申込締切日前まで表示
- 締切後はトップから自動で非表示
- 記事ページには「募集は終了しました」と表示
- 実施後に関連活動レポートIDを設定できる
- 関連活動レポートがある場合は「開催後の様子はこちら」とリンク表示

申込は外部Formsリンクを使うため、application_urlを設定できるようにしてください。

表形式を基本にしてください。将来的にカード表示へ切り替えられる構成にしてください。

入力項目：

- 講師名
- 講師肩書き

- 参加者の声
- Instagram用本文

- 短いメモ

通常記事と同じ見た目で公開します。

## 11. ChatGPTコピペ運用

初期段階ではAI API連携は不要です。  
管理画面内に以下を設置してください。

- ChatGPT用メモを生成する欄
- コピーするボタン
- Web記事文貼り戻し欄
- Instagram本文貼り戻し欄
- ハッシュタグ欄

「ChatGPT用メモ」は、入力内容をもとに以下の形式で生成してください。

```text
あなたはJAしまね雲南地区本部の講座サイト用ライターです。
以下の素材から、Web記事本文、抜粋文、Instagram本文、ハッシュタグを作成してください。

条件：
- Web記事は暮らしの読み物風
- 300〜500字程度
- 硬すぎず、軽すぎない文章
- Instagramは少し短く親しみやすく
- 絵文字は控えめ
- 写真掲載や個人情報に配慮

素材：
講座名：...
開催日：...
タイトル：...
講師：...
内容メモ：...
参加者の声：...
```

### 写真相談プロンプト（現行プロトタイプで追加）

記事文用のメモとは別に、写真を相談するためのプロンプトもコピーできるようにしてください。記事情報（講座名・開催日・タイトル）と写真枚数を差し込み、次を確認する内容にします。

- メイン写真に向いている写真
- ギャラリー向きの写真
- 使わない方がよい写真
- 顔や個人情報が写りすぎていないか
- トリミングするなら中央・上寄せ・下寄せ・左寄せ・右寄せのどれがよいか
- 各写真の短いキャプション
- 記事全体として写真が足りているか

AIの回答は貼り付け欄で受け取り、自動反映はせず、職員が写真ごとの設定（役割・トリミング位置・キャプション等）に手動で反映します。

## 12. 公開前チェック

公開前に以下の3点チェックをすべて必須にしてください（旧設計の1点から拡張）。

```text
□ 写真掲載に問題がないことを確認しました
□ 顔や個人情報が写りすぎていないことを確認しました
□ 必要に応じて掲載許可を確認しました
```

3点すべてにチェックが入るまで、公開できないようにしてください。

講座ごとに注意文を変えてください。

赤ちゃん食堂・アグリスクール：

```text
赤ちゃん・子ども・保護者の顔が分かる写真は原則避けてください。
料理、手元、後ろ姿、会場風景中心の写真を推奨します。
```

その他講座：

```text
参加者の顔が分かる写真を掲載する場合は、掲載許可を確認してください。
料理、作品、手元、会場風景中心の写真を推奨します。
```

## 13. 状態管理

記事の状態は以下の4つです。

- draft：下書き
- scheduled：公開予約
- published：公開
- private：非公開

公開予約日時が現在日時を過ぎたら、公開扱いにしてください。  
GASの時間主導トリガーで状態を更新するか、表示時に判定してください。

## 15. Googleドライブ構成

「うんなん暮らしの学び帖」に関して Drive につくるものは、すべて Drive 直下の
「うんなん暮らしの学び帖」フォルダの中に入れてください（写真・サムネイル・
バックアップ・エクスポート・スプレッドシート本体も含む）。

```text
うんなん暮らしの学び帖   ← Drive直下のルート。関連ファイルは全部この中
├─ （スプレッドシート本体）
├─ images
│  ├─ reports
│  ├─ courses
│  └─ thumbnails
├─ backups
└─ exports
```

写真はアップロード時にこのルート配下（images/reports など）へ保存し、file_idを
ImagesシートまたはArticlesシートに保存してください。GAS 側は
`getRootFolder_()` / `getFolder_([...])` でこのフォルダ配下を get-or-create します。

## 21. 重要な実装方針

- 職員の負担を増やさない
- 必須項目を少なくする
- 写真が少なくても崩れない
- 募集がない時にサイトが古く見えない
- 活動レポートを主役にする
- 初期はシンプルに作り、将来拡張できるようにする
