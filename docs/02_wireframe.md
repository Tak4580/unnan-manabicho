# 画面設計・ワイヤーフレーム

> 本書は当初の画面設計をもとにしつつ、現行のHTML試作品（`index.html` / `style.css` / `app.js`）の構成に合わせて更新している。当初から変わった点は「現行プロトタイプ」として補足する。主な変更：ヘッダーは5項目（ホーム／活動レポート／講座一覧／Instagram／お問い合わせ）、募集はトップ横の控えめな「次回のご案内」、レポート一覧に検索・年の絞り込み、記事詳細にサブタイトル・開催場所・共有ボタン、管理画面はドラッグ可能なフローティング「Phone View」プレビュー、写真管理と3点チェック、お問い合わせページの追加。

## 1. 前提

本サイトはスマホ閲覧を主に想定する。PCでも閲覧できるが、基本はスマホで縦に読みやすい構成とする。

デザインの方向性：

- 暮らしの雑誌風
- 地域ニュースサイト風
- 少し手作り感
- 白基調
- 余白多め
- 講座ごとに差し色
- 写真がなくても成立するカード設計

### 現行プロトタイプの補足

- スマホプレビューは、画面上をドラッグで動かせるフローティングパネル「Phone View」（閉じる・最小化・最大化つき）として実装している。固定の右カラムに限定しない。
- 入力フォームの中に写真管理を組み込む：撮影ガイド／複数アップロード／「掲載イメージ・元写真」切り替え／写真ごとの役割・トリミング位置・キャプション・alt・チェック／AI写真相談プロンプトのコピー／掲載前3点チェック。

```text
┌─ 写真管理 ─────────────────┐
│ [AIに写真を相談するプロンプトをコピー] │
│ 撮影ガイド（全体・手元・完成品 …）  │
│ [写真を選択（複数可）]              │
│ 表示：[掲載イメージ] [元写真]        │
│ ┌ 写真1 ─────────────┐         │
│ │ 使用/役割/トリミング位置/alt   │         │
│ │ キャプション / AIアドバイス    │         │
│ └───────────────────┘         │
│ □写真掲載OK □個人情報OK □掲載許可 │
└────────────────────────────┘
```

## 13. 管理画面：公開前チェック

公開ボタンの近くに、必須の3点チェックを置く。

```text
□ 写真掲載に問題がないことを確認しました
□ 顔や個人情報が写りすぎていないことを確認しました
□ 必要に応じて掲載許可を確認しました
```

3点すべてにチェックが入るまで公開不可。

講座ごとに注意文を変える。

赤ちゃん食堂・アグリスクール：

```text
赤ちゃん・子ども・保護者の顔が分かる写真は原則避けてください。
料理、手元、後ろ姿、会場風景中心の写真を推奨します。
```

その他講座：

```text
参加者の顔が分かる写真を掲載する場合は、掲載許可を確認してください。
料理、作品、手元、会場風景中心の写真を推奨します。
```
